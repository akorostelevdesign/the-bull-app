# Requirements Document

## Introduction

Система допродаж (upsell) для приложения официантов ресторана "The Bull". Когда официант добавляет блюдо в заказ на странице стола, система автоматически показывает релевантные рекомендации в виде чипов/кнопок прямо под карточкой блюда. Нажатие на чип добавляет рекомендованную позицию в тот же заказ. Правила допродаж настраиваются в разделе настроек — каждое правило можно включить или выключить. Настройки хранятся в localStorage.

## Glossary

- **UpsellRule**: Правило допродажи — объект, связывающий триггер (блюдо или категорию) со списком рекомендаций.
- **UpsellChip**: Кнопка-чип, отображаемая под карточкой блюда с предложением добавить рекомендованную позицию.
- **UpsellSettings**: Объект настроек, хранящий состояние включения/выключения каждого правила допродажи.
- **Trigger**: Условие активации правила — совпадение по имени блюда или категории.
- **Recommendation**: Позиция меню, предлагаемая к добавлению при срабатывании правила.
- **UpsellSystem**: Система допродаж в целом, включающая данные, UI и настройки.
- **OrderContext**: React-контекст, управляющий состоянием заказов по столам.
- **Table_Page**: Страница стола (Table.jsx), где официант управляет заказом.
- **Settings_Page**: Страница настроек курсов (CourseSettings.jsx), расширяемая разделом допродаж.

## Requirements

### Requirement 1: Данные правил допродаж

**User Story:** As a разработчик, I want иметь централизованный файл с правилами допродаж, so that правила легко поддерживать и расширять.

#### Acceptance Criteria

1. THE UpsellSystem SHALL хранить все правила допродаж в отдельном файле `src/data/upsell-rules.js`.
2. WHEN правило определяется по категории блюда, THE UpsellSystem SHALL применять его ко всем блюдам этой категории.
3. WHEN правило определяется по точному имени блюда, THE UpsellSystem SHALL применять его только к этому блюду.
4. THE UpsellSystem SHALL содержать правила как минимум для следующих триггеров:
   - Категории стейков (Прайм стейки, Альтернативные стейки, Хоспер): рекомендовать прожарку (если не выбрана), соус, гарнир, красное вино (Мальбек или Каберне Совиньон).
   - Вителло тоннато: рекомендовать Фокаччу с розмарином.
   - Тартар из говядины: рекомендовать яйцо перепелиное, гренки, чипсы из бекона.
   - Коктейли (категория): предлагать двойную порцию.
   - Чай (категория): рекомендовать мёд, мяту, чабрец, сахар, лимон.
   - Пиво (категория): рекомендовать закуски к пиву (Чесночные гренки, Крылья куриные, Куриные стрипсы).
   - Бургеры (категория): рекомендовать картофель фри, газированный напиток.
   - Десерты (категория): рекомендовать кофе или чай.
5. EACH правило SHALL иметь уникальный идентификатор (`id`), человекочитаемое название (`label`) и список рекомендаций (`recommendations`).

### Requirement 2: Отображение чипов допродаж в карточке блюда

**User Story:** As a официант, I want видеть подсказки допродаж прямо под карточкой блюда, so that я могу быстро предложить гостю дополнительные позиции.

#### Acceptance Criteria

1. WHEN официант добавляет блюдо в заказ, THE Table_Page SHALL отображать чипы допродаж под карточкой этого блюда, если для него существует активное правило.
2. WHEN для блюда нет активных правил допродаж, THE Table_Page SHALL не отображать блок чипов под карточкой.
3. THE UpsellChip SHALL отображать название рекомендованной позиции в виде кнопки-чипа.
4. WHEN официант нажимает на UpsellChip, THE Table_Page SHALL добавить рекомендованную позицию в заказ того же стола.
5. WHEN рекомендованная позиция уже присутствует в заказе стола, THE UpsellChip SHALL отображаться в состоянии "добавлено" (визуально отличаться).
6. WHILE блюдо находится в заказе со статусом "served" (вынесен), THE Table_Page SHALL скрывать чипы допродаж для этого блюда.
7. THE UpsellChip SHALL быть достаточно компактным, чтобы не нарушать существующий layout карточки блюда.

### Requirement 3: Настройки допродаж

**User Story:** As a официант, I want управлять правилами допродаж в настройках, so that я могу отключить нерелевантные предложения.

#### Acceptance Criteria

1. THE Settings_Page SHALL содержать раздел "Допродажи" с переключателями для каждого правила.
2. WHEN официант переключает правило в настройках, THE UpsellSystem SHALL немедленно применить изменение — чипы для отключённого правила перестают отображаться.
3. THE UpsellSystem SHALL отображать для каждого правила его `label` (название) и краткое описание триггера.
4. IF правило отключено, THEN THE Table_Page SHALL не отображать чипы, связанные с этим правилом.
5. THE UpsellSystem SHALL по умолчанию включать все правила допродаж при первом запуске.

### Requirement 4: Хранение настроек

**User Story:** As a официант, I want чтобы мои настройки допродаж сохранялись между сессиями, so that мне не нужно настраивать их каждый раз заново.

#### Acceptance Criteria

1. WHEN официант изменяет настройки допродаж, THE UpsellSystem SHALL сохранить состояние в localStorage под ключом `upsell-settings`.
2. WHEN приложение загружается, THE UpsellSystem SHALL восстановить настройки допродаж из localStorage.
3. IF в localStorage отсутствуют сохранённые настройки, THEN THE UpsellSystem SHALL использовать настройки по умолчанию (все правила включены).
4. WHEN в localStorage хранятся настройки для устаревших правил (которых больше нет в `upsell-rules.js`), THE UpsellSystem SHALL игнорировать их без ошибок.
