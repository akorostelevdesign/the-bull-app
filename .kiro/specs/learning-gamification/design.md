# Дизайн-документ: Learning Gamification

## Архитектура

Модуль встраивается в существующее React + Vite PWA. Всё состояние хранится в `localStorage` (ключ `bull_learn_v3`). Никаких новых зависимостей — только существующие `lucide-react` и `react`.

### Структура файлов

```
src/pages/Learning.jsx          — главный компонент (роутинг экранов)
src/pages/Learning.css          — стили
src/data/learning-content.js    — весь контент (сервис, знания, симуляции)
src/hooks/useProgress.js        — хук для работы с Progress_Store
```

## Экраны и навигация

```
Onboarding (3 шага)
  └─ Главный экран (Home)
       ├─ Сервис → выбор режима → упражнение
       ├─ Кухня  → выбор режима → упражнение
       ├─ Бар    → выбор режима → упражнение
       ├─ Знания → выбор режима → упражнение
       └─ Доп. продажи (разблокируется после Кухни)
```

Навигация через `screen` state в корневом компоненте — без React Router (всё на одной странице `/learning`).

## Онбординг (3 шага)

### Шаг 1 — Срок курса
4 карточки: 7 / 14 / 28 / 45 дней. Под каждой — подсказка (например, «~15 мин/день»).

### Шаг 2 — Режим
- «Я учусь» — начисляются XP, активны жизни и дедлайн
- «Я практикуюсь» — без XP, без ограничений

### Шаг 3 — Грейд
- Грейд 1: составы + аллергены + безалкогольный бар
- Грейд 2: весь контент модуля

После шага 3 — сохранение в Progress_Store, переход на Home.

| Уровень | Название       | Порог XP |
|---------|----------------|----------|
| 1       | Новичок        | 0        |
| 2       | Стажёр         | 300      |
| 3       | Официант       | 700      |
| 4       | Гриль Мастер   | 1 500    |
| 5       | Профи          | 3 000    |
| 6       | Легенда        | 6 000    |

**Начисление XP:**
- Правильный ответ (обычное блюдо): +10 XP
- Правильный ответ (сложное блюдо / маринад): +20 XP
- Завершение раздела: +100 XP
- Правильный ответ в Battle: +15 XP
- Правильный ответ в Order_Simulation с доп. продажей: +25 XP
- Режим «Я практикуюсь»: 0 XP

## Раздел «Сервис»

### Экран выбора режима
3 кнопки: «Изучить», «Собери чек-лист», «Ticket-time».

### Режим «Изучить»
Слайдер из 6 шагов. Каждый шаг — карточка с заголовком и списком действий. Свайп влево/вправо между шагами.

### Механика «Собери чек-лист»
- Показывается название шага (например, «Расчёт»)
- Внизу — перемешанные карточки действий (из всех 6 шагов)
- Пользователь тапает нужные → они перемещаются в зону шага
- Подтверждение → подсветка правильных/неправильных → XP

### Тренажёр «Ticket-time»
Серия из 8–10 вопросов:
> «Принесли напитки через 8 минут. Это ок?»
> Кнопки: ✅ Да / ❌ Нет
После ответа — пояснение (стандарт: 5–7 минут → Нет).

## Раздел «Кухня»

### Экран выбора режима
5 кнопок: Каталог / Флэш-карточки / Квиз / Битва / Симуляция заказа.

### Каталог
Скроллируемый список блюд. Тап → детальная карточка: фото, состав, **аллергены** (выделены красным), цена, код.

### Флэш-карточки
Лицевая: название блюда + фото.
Обратная: состав + аллергены (жирным).
Кнопки: «Знаю» (+10/+20 XP) / «Не помню» (+2 XP).

### Квиз
Вопросы двух типов:
1. «Какой соус в [блюде]?» — 4 варианта
2. «Найди ошибку в составе» — показывается состав с одной заменой, нужно найти

### Битва за состав
- Показывается описание/аллергены блюда
- Таймер 15 секунд
- Нужно выбрать название из 4 вариантов
- Чем быстрее — тем больше XP (от 5 до 15 XP)

### Симуляция заказа (дерево ответов)
Пример сценария:
```
Гость: «Стейк без крови, пожалуйста»
├─ «В стейке нет крови, это миоглобин» → +25 XP ⭐
├─ «Сделаем Well Done» → +10 XP
└─ «Всё мясо свежее» → +0 XP
```
После выбора — пояснение почему этот ответ лучше/хуже.

## Раздел «Напитки и бар»

Аналогичная структура: Каталог / Флэш-карточки / Битва / Симуляция.
Флэш-карточки: лицевая — название, обратная — состав + **происхождение** (жирным).
Грейд 1: только безалкогольные напитки.

## Раздел «Знания»

### Темы и механики

| Тема | Механика |
|------|----------|
| Степени прожарки | Интерактивный слайдер |
| Зерно vs Трава | Свайп-карточки (Tinder) |
| LAST / LATTE | Drag-and-drop порядка шагов |
| Правда или Миф | Quiz Да/Нет с пояснением |
| Остальные темы | Флэш-карточки + Quiz |

### Слайдер прожарок
Горизонтальная полоска с 6 точками (Blue → Well Done).
Под полоской: название, температура, описание цвета сока.
Задание: «Поставь на Medium Rare» → попасть в диапазон 55–57°C.

### Свайп «Зерно vs Трава»
Карточка с характеристикой (например, «Мягкий, сочный, 210 дней откорма»).
Свайп вправо = Зерно, влево = Трава.
Правильно → +10 XP, неправильно → показать правильный ответ.

### LAST / LATTE
Показываются перемешанные шаги. Пользователь перетаскивает в правильный порядок.
Подтверждение → анимация правильного порядка → XP.

## Раздел «Доп. продажи» (Паровоз)

Разблокируется когда `sectionProgress.kitchen === 100`.

### Механика
1. Показывается блюдо гостя (например, «Тартар из говядины»)
2. Пользователь выбирает доп. позиции из списка (quickSelect из db.js)
3. Каждая добавленная позиция = +5 XP
4. Показывается итоговая сумма цепочки и максимально возможная
5. Если набрал >80% от максимума → бонус +20 XP

## Система жизней

- Старт курса: 3 жизни (❤️❤️❤️)
- Пропущен день → -1 жизнь (проверяется при открытии приложения)
- 0 жизней → рестарт курса (Onboarding_Flow заново)
- Восстановить 1 жизнь → -200 XP (кнопка в хедере при наличии XP)
- В режиме «Я практикуюсь» жизни не расходуются

## Колесо Фортуны

Показывается один раз после завершения курса.

### Сектора
| Приз | Вероятность |
|------|-------------|
| 90 дней подписки | 1% |
| 60 дней подписки | 4% |
| 30 дней подписки | 10% |
| Заморозка стрика (7 дней) | 25% |
| +500 XP | 30% |
| Ачивка «Легенда» | 30% |

### Реализация
CSS-анимация вращения колеса (`transform: rotate`). Угол рассчитывается заранее на основе случайного результата с заданными вероятностями. Анимация 3–4 секунды с замедлением (`cubic-bezier`).

## Порционная выдача контента

При старте курса весь контент делится на `courseDays` равных порций.
Каждый день открывается следующая порция.
Логика в `useProgress.js`:

```js
function getTodayPortion(allItems, courseDays, courseStartDate) {
  const daysPassed = Math.floor((Date.now() - new Date(courseStartDate)) / 86400000);
  const dayIndex = Math.min(daysPassed, courseDays - 1);
  const chunkSize = Math.ceil(allItems.length / courseDays);
  return allItems.slice(0, (dayIndex + 1) * chunkSize);
}
```
